# Database Schema

This document summarizes the SQLAlchemy database schema in a compact form for use as
AI-agent context and as a quick human reference. It is generated by hand from the models
under `app/core/db/models/` - keep it in sync when models change.

## Schemas

- `core` - central business domain (party/person/company, students, tutors, subjects)
- `geo` - geographic reference data (PLZ/Ort)
- `ext` - links from external system identifiers to a `core.party`
- `bot` - SkillBot operational state (Discord guilds/channels/users, workspaces, permissions, job queue, two-phase operations)
- `auth` - OAuth2 application clients, secrets, scopes, and the auth audit trail

## Shared Columns

Models using `TimestampMixin` include:

```sql
created_at TIMESTAMPTZ NOT NULL DEFAULT now()
updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
```

Conventions used below:

- `DEFAULT uuid4` denotes an **application-side** default (`uuid.uuid4`); UUID primary keys
  have no SQL server default.
- `TEXT` covers both SQLAlchemy `Text` and unbounded `String` columns - Postgres renders the
  latter as `VARCHAR`, which it treats identically to `TEXT`.

```sql
-- core.party
id UUID PRIMARY KEY DEFAULT uuid4

updated_at TIMESTAMPTZ NOT NULL DEFAULT now()

-- core.person
party_id UUID PRIMARY KEY REFERENCES core.party(id) ON DELETE CASCADE
firstname TEXT NOT NULL
lastname TEXT NOT NULL
created_at TIMESTAMPTZ NOT NULL DEFAULT now()
updated_at TIMESTAMPTZ NOT NULL DEFAULT now()

-- core.company
party_id UUID PRIMARY KEY REFERENCES core.party(id) ON DELETE CASCADE
name TEXT NOT NULL
created_at TIMESTAMPTZ NOT NULL DEFAULT now()
updated_at TIMESTAMPTZ NOT NULL DEFAULT now()

-- core.student
person_id UUID PRIMARY KEY REFERENCES core.person(party_id) ON DELETE CASCADE
preferred_meeting_tool preferred_meeting_tool NOT NULL
created_at TIMESTAMPTZ NOT NULL DEFAULT now()
updated_at TIMESTAMPTZ NOT NULL DEFAULT now()

-- core.tutor
person_id UUID PRIMARY KEY REFERENCES core.person(party_id) ON DELETE CASCADE
created_at TIMESTAMPTZ NOT NULL DEFAULT now()
updated_at TIMESTAMPTZ NOT NULL DEFAULT now()

-- core.subject
id INTEGER PRIMARY KEY AUTOINCREMENT
title TEXT NOT NULL

-- core.student_subject
student_id UUID REFERENCES core.student(person_id) ON DELETE CASCADE
subject_id INTEGER REFERENCES core.subject(id) ON DELETE CASCADE
PRIMARY KEY (student_id, subject_id)

-- core.tutor_subject
tutor_id UUID REFERENCES core.tutor(person_id) ON DELETE CASCADE
subject_id INTEGER REFERENCES core.subject(id) ON DELETE CASCADE
PRIMARY KEY (tutor_id, subject_id)

-- core.contact_info
id UUID PRIMARY KEY DEFAULT uuid4
party_id UUID NOT NULL REFERENCES core.party(id) ON DELETE CASCADE
type contact_info_type NOT NULL
value TEXT NOT NULL
label TEXT NULL
created_at TIMESTAMPTZ NOT NULL DEFAULT now()
updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
UNIQUE (party_id, type, value) -- uq_contact_info
INDEX (party_id)

-- core.party_relation
from_party_id UUID REFERENCES core.party(id) ON DELETE CASCADE
to_party_id UUID REFERENCES core.party(id) ON DELETE CASCADE
type party_relation_type NOT NULL
created_at TIMESTAMPTZ NOT NULL DEFAULT now()
updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
PRIMARY KEY (from_party_id, to_party_id, type)
```

## Relationship Model

- `core.party` is the central entity.
- A `party` is either a `person` or a `company`, represented by `party.type`.
- `core.person.party_id` is both primary key and foreign key to `core.party.id`.
- `core.company.party_id` is both primary key and foreign key to `core.party.id`.
- `core.student` and `core.tutor` are roles of a `person`.
- `core.student_subject` and `core.tutor_subject` are many-to-many join tables between students/tutors and subjects.
- `core.contact_info` belongs to any `party`, so it can attach to both persons and companies.
- `core.party_relation` models directed relationships between parties, such as `parent_of`, `tutor_of`, and `pays_for`.
- Each `ext.*` table links an external system identifier to a `party`. The non-Discord links are
  one-to-one (`party_id` UNIQUE); `ext.discord_account` allows several per party, guarded by a
  partial unique of at most one `is_primary AND active` account.
- Most foreign keys use `ON DELETE CASCADE`.

## Bot Schema

The `bot` schema holds SkillBot's operational state. Discord snowflake ids are stored as
`BIGINT` (not auto-incremented). See [ADR 0003](decisions/0003-two-phase-transitions.md) and
[ADR 0004](decisions/0004-forge-first-job-queue.md) for the operation/job design.

### Enums


### Discord Topology

```sql
-- bot.discord_guild
guild_id BIGINT PRIMARY KEY  -- Discord snowflake, not auto-incremented
name TEXT NULL
is_primary BOOLEAN NOT NULL DEFAULT false
active BOOLEAN NOT NULL DEFAULT true
created_at TIMESTAMPTZ NOT NULL DEFAULT now()
updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
-- partial unique: at most one primary+active guild
UNIQUE (is_primary) WHERE is_primary AND active  -- uq_discord_guild_primary_active

-- bot.discord_channel
channel_id BIGINT PRIMARY KEY
guild_id BIGINT NOT NULL REFERENCES bot.discord_guild(guild_id) ON DELETE CASCADE
parent_channel_id BIGINT NULL  -- self-ref via (guild_id, parent_channel_id), ON DELETE SET NULL
type discord_channel_type NOT NULL
name TEXT NULL
managed_by_bot BOOLEAN NOT NULL DEFAULT true
deleted_at TIMESTAMPTZ NULL  -- soft delete
created_at TIMESTAMPTZ NOT NULL DEFAULT now()
updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
UNIQUE (guild_id, channel_id)
INDEX (guild_id, deleted_at), INDEX (parent_channel_id)

-- bot.discord_user
discord_id BIGINT PRIMARY KEY
role member_role NOT NULL
nick_name TEXT NOT NULL  -- CHECK (nick_name <> '')
active BOOLEAN NOT NULL DEFAULT true
created_at TIMESTAMPTZ NOT NULL DEFAULT now()
updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
INDEX (role, active)

-- bot.discord_role_binding  (maps a member_role to a Discord guild role)
guild_id BIGINT REFERENCES bot.discord_guild(guild_id) ON DELETE CASCADE
member_role member_role
role_id BIGINT NULL
role_name TEXT NOT NULL
active BOOLEAN NOT NULL DEFAULT true
created_at TIMESTAMPTZ NOT NULL DEFAULT now()
updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
PRIMARY KEY (guild_id, member_role)
```

### Workspaces

```sql
-- bot.tutor_workspace  (one Discord category + command channel per tutor per guild)
guild_id BIGINT REFERENCES bot.discord_guild(guild_id) ON DELETE CASCADE
tutor_discord_id BIGINT REFERENCES bot.discord_user(discord_id) ON DELETE CASCADE
category_channel_id BIGINT NOT NULL  -- FK (guild_id, category_channel_id) -> discord_channel, ON DELETE RESTRICT
command_channel_id BIGINT NOT NULL   -- FK (guild_id, command_channel_id) -> discord_channel, ON DELETE RESTRICT
student_channel_capacity SMALLINT NOT NULL DEFAULT 49  -- CHECK BETWEEN 0 AND 49
created_at TIMESTAMPTZ NOT NULL DEFAULT now()
updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
PRIMARY KEY (guild_id, tutor_discord_id)
UNIQUE (category_channel_id), UNIQUE (command_channel_id)

-- bot.archive_category  (overflow categories holding stashed student channels)
guild_id BIGINT REFERENCES bot.discord_guild(guild_id) ON DELETE CASCADE
archive_no INTEGER  -- CHECK (> 0)
category_channel_id BIGINT NOT NULL  -- FK -> discord_channel, ON DELETE RESTRICT
capacity SMALLINT NOT NULL DEFAULT 50  -- CHECK BETWEEN 1 AND 50
created_at TIMESTAMPTZ NOT NULL DEFAULT now()
updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
PRIMARY KEY (guild_id, archive_no)
UNIQUE (category_channel_id)
UNIQUE (guild_id, category_channel_id)
INDEX (guild_id)

-- bot.student_workspace  (a student's channel; lives in tutor's category or in an archive)
guild_id BIGINT REFERENCES bot.discord_guild(guild_id) ON DELETE CASCADE
student_discord_id BIGINT REFERENCES bot.discord_user(discord_id) ON DELETE CASCADE
tutor_discord_id BIGINT NOT NULL REFERENCES bot.discord_user(discord_id) ON DELETE RESTRICT
channel_id BIGINT NULL  -- FK -> discord_channel, ON DELETE SET NULL; UNIQUE
channel_state student_channel_state NOT NULL DEFAULT 'tutor_category'
current_parent_channel_id BIGINT NULL  -- FK -> discord_channel, ON DELETE RESTRICT
archive_category_channel_id BIGINT NULL  -- FK -> archive_category, ON DELETE RESTRICT
stashed_at TIMESTAMPTZ NULL
popped_at TIMESTAMPTZ NULL
created_at TIMESTAMPTZ NOT NULL DEFAULT now()
updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
PRIMARY KEY (guild_id, student_discord_id)
INDEX (guild_id, tutor_discord_id)
INDEX (guild_id, tutor_discord_id, channel_state)
INDEX (guild_id, channel_state)
INDEX (archive_category_channel_id, channel_state)
-- CHECKs keep state coherent: archive_category set iff channel_state='archive_category',
-- archive state requires channel_id and parent = archive_category_channel_id.
```

### Permissions

```sql
-- bot.permission_group
key TEXT PRIMARY KEY
name TEXT NOT NULL
active BOOLEAN NOT NULL DEFAULT true
created_at/updated_at TIMESTAMPTZ NOT NULL DEFAULT now()

-- bot.discord_user_permission_group  (user <-> group membership)
discord_id BIGINT REFERENCES bot.discord_user(discord_id) ON DELETE CASCADE
group_key TEXT REFERENCES bot.permission_group(key) ON DELETE CASCADE
created_at TIMESTAMPTZ NOT NULL DEFAULT now()
PRIMARY KEY (discord_id, group_key)

-- bot.permission_grant  (allow/deny an action for a role/group/user, with priority)
id UUID PRIMARY KEY DEFAULT uuid4
subject_type permission_subject_type NOT NULL  -- role | group | user
subject_key TEXT NOT NULL
action_key TEXT NOT NULL
effect permission_grant_effect NOT NULL  -- allow | deny
priority INTEGER NOT NULL DEFAULT 0
active BOOLEAN NOT NULL DEFAULT true
created_at/updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
UNIQUE (subject_type, subject_key, action_key)
INDEX (subject_type, subject_key, active)
INDEX (action_key)

-- bot.command_env_channel  (which channels a command kind is allowed in)
guild_id BIGINT REFERENCES bot.discord_guild(guild_id) ON DELETE CASCADE
channel_id BIGINT  -- FK (guild_id, channel_id) -> discord_channel, ON DELETE CASCADE
kind command_env_kind  -- admin_cmd | tutor_cmd
owner_discord_id BIGINT NULL REFERENCES bot.discord_user(discord_id) ON DELETE CASCADE
created_at/updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
PRIMARY KEY (guild_id, channel_id, kind)
INDEX (guild_id, kind)
INDEX (guild_id, owner_discord_id, kind)
-- partial UNIQUE (guild_id, owner_discord_id, kind) WHERE owner_discord_id IS NOT NULL
```

### Queue & Operations

```sql
-- bot.job  (Forge-first work queue; SkillBot claims via SELECT ... FOR UPDATE SKIP LOCKED)
job_id UUID PRIMARY KEY DEFAULT uuid4
kind TEXT NOT NULL
payload JSONB NOT NULL DEFAULT '{}'
status job_status NOT NULL DEFAULT 'pending'
attempt SMALLINT NOT NULL DEFAULT 0       -- CHECK (>= 0)
max_attempts SMALLINT NOT NULL DEFAULT 5  -- CHECK (>= 1)
available_at TIMESTAMPTZ NOT NULL DEFAULT now()
claimed_at TIMESTAMPTZ NULL
claimed_by TEXT NULL
completed_at TIMESTAMPTZ NULL
failed_at TIMESTAMPTZ NULL
last_error TEXT NULL
created_at/updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
INDEX (available_at) WHERE status='pending'  -- ix_job_claimable
INDEX (kind, status)

-- bot.operation  (two-phase prepare/commit reservation log; intentionally has no FKs)
operation_id UUID PRIMARY KEY DEFAULT uuid4
kind operation_kind NOT NULL
status operation_status NOT NULL DEFAULT 'prepared'
guild_id BIGINT NOT NULL
subject_discord_id BIGINT NOT NULL
tutor_discord_id BIGINT NULL
reserved_archive_category_channel_id BIGINT NULL  -- slot reserved by a prepared stash
plan JSONB NOT NULL DEFAULT '{}'
expires_at TIMESTAMPTZ NOT NULL  -- prepared rows expire after OPERATION_TTL (10 min)
committed_at TIMESTAMPTZ NULL
failed_at TIMESTAMPTZ NULL
last_error TEXT NULL
created_at/updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
INDEX (status, expires_at)
INDEX (guild_id, subject_discord_id, status)
-- partial: counts outstanding stash reservations per archive category
INDEX (guild_id, reserved_archive_category_channel_id) WHERE status='prepared' AND reserved_archive_category_channel_id IS NOT NULL  -- ix_operation_reservation

-- bot.app_command_audit_log  (per-command permission decision + error trail)
id UUID PRIMARY KEY DEFAULT uuid4
guild_id / channel_id / discord_id BIGINT NULL
command_name TEXT NOT NULL
permission_action / permission_source / permission_reason TEXT NULL
permission_allowed BOOLEAN NULL
error_type / error TEXT NULL
created_at TIMESTAMPTZ NOT NULL DEFAULT now()
INDEX (created_at), INDEX (guild_id, command_name, created_at), INDEX (discord_id, created_at)
```

## Auth Schema

OAuth2 client-credentials. Clients authenticate with a `client_id` + secret, are granted
scopes, and receive JWTs. See [ADR 0001](decisions/0001-openapi-as-contract.md) for the API
contract and `app/core/auth/` for the runtime.

```sql
auth.application_client_status = ('active', 'disabled')

-- auth.application_client
id UUID PRIMARY KEY DEFAULT uuid4
client_id TEXT NOT NULL UNIQUE
name TEXT NOT NULL
description TEXT NULL
status application_client_status NOT NULL DEFAULT 'active'
created_at/updated_at TIMESTAMPTZ NOT NULL DEFAULT now()

-- auth.application_client_secret  (Argon2-hashed; supports rotation/expiry/revocation)
id UUID PRIMARY KEY DEFAULT uuid4
application_client_id UUID NOT NULL REFERENCES auth.application_client(id) ON DELETE CASCADE
secret_hash TEXT NOT NULL
label TEXT NULL
expires_at / last_used_at / revoked_at TIMESTAMPTZ NULL
created_at TIMESTAMPTZ NOT NULL DEFAULT now()
INDEX (application_client_id)

-- auth.permission_scope  (e.g. bot:read, bot:write, auth:clients:manage)
key TEXT PRIMARY KEY
description TEXT NOT NULL
active BOOLEAN NOT NULL DEFAULT true
created_at/updated_at TIMESTAMPTZ NOT NULL DEFAULT now()

-- auth.application_client_scope_grant  (client <-> scope)
application_client_id UUID REFERENCES auth.application_client(id) ON DELETE CASCADE
scope_key TEXT REFERENCES auth.permission_scope(key) ON DELETE RESTRICT
created_at TIMESTAMPTZ NOT NULL DEFAULT now()
PRIMARY KEY (application_client_id, scope_key)

-- auth.auth_audit_log
id UUID PRIMARY KEY DEFAULT uuid4
principal_type / principal_id TEXT NULL
event_type TEXT NOT NULL
success BOOLEAN NOT NULL
detail TEXT NULL
created_at TIMESTAMPTZ NOT NULL DEFAULT now()
INDEX (created_at), INDEX (principal_type, principal_id, created_at)
```
